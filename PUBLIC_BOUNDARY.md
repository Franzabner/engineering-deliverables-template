# Public Boundary

## Public-Safe Content

- Synthetic README, STATUS, CLAIMS, and PUBLIC_BOUNDARY templates.
- Synthetic BOM structure templates.
- Generic control narrative, commissioning, simulation report, model card, dataset card, and handoff checklist templates.
- Mermaid deliverable review flow diagrams.

## Private / Sealed / Held Content

This repo must not contain stamped-engineering implications, certification claims, code-compliance approval claims, legal approval, customer deliverables, customer data, Foundation-private data, secrets, credentials, private URLs, production procedures, production BOMs, production CAD, production schematics, sealed source, private model artifacts, private logs, active client deliverables, active service offerings, pricing, turnaround promises, released model claims, released dataset claims, Space release claims, or internal company product names.

## Review Rule

Any move from local scaffold to public GitHub repo requires human review, validator pass, boundary/status scan, clean `git diff --check`, clean `git status --short`, and separate approval for routing.
