# PCB Review Report Template

Status: scaffolded
Publication state: public scaffold
Release state: not released

## Purpose

This template provides a public-safe report shape for PCB review notes. It is a documentation aid, not an approval or stamped deliverable.

## Report Header

| Field | Entry rule |
| --- | --- |
| Artifact status | Use planned, scaffolded, public scaffold, or not released. |
| Review scope | Describe generic areas reviewed, not customer files or production source. |
| Evidence source | Use synthetic examples or reviewed public-safe material. |
| Boundary status | State public, private, sealed, or mixed. |
| Human review need | List unresolved review questions. |

## Finding Fields

| Field | Entry rule |
| --- | --- |
| Category | Power, signal, documentation, boundary, or evidence. |
| Severity | Informational, needs review, hold private, or escalation. |
| Observation | Public-safe wording only. |
| Held material | Name the category of material held out, not the private detail. |

## Exclusions

Do not include production board source, KiCad production board claims, Gerbers, BOMs, exact pin maps, customer data, manufacturing instructions, PCB manufacturing approval claims, certification claims, sign-off claims, deployment claims, production-readiness claims, or active client deliverable claims.
